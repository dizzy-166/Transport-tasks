Основные улучшения:
1. Принцип DRY (Don't Repeat Yourself)

    Было: Дублирование логики ввода/вывода и обработки данных

    Стало: Выделены общие классы и методы для повторного использования

2. Принцип единственной ответственности (SRP)

    Было: Весь код в одном классе со статическими методами

    Стало: Разделение на логические блоки:

        TransportData - только хранение и валидация данных

        TransportResult - только представление результата

        NorthWestCornerStrategy и MinCostStrategy - только реализация алгоритмов

        TransportProblemException - только обработка ошибок

3. Стратегия (Strategy Pattern)

    Введен интерфейс ITransportStrategy для унификации алгоритмов

    Каждый метод решения реализует этот интерфейс независимо

4. Обработка исключений

    Было: Возврат статусных сообщений (minRow == -1)

    Стало: Генерация пользовательского исключения TransportProblemException

5. Инкапсуляция и безопасность

    Все классы sealed (запрет наследования)

    Члены классов private где возможно

    Валидация данных в конструкторе

6. Документирование

    Добавлены XML-комментарии для всех публичных членов

    Понятные названия классов и методов

7. Чистота кода

    Убраны вложенные циклы (вынесены в отдельные методы)

    Добавлены фигурные скобки для всех конструкций

    Улучшена читаемость за счет выделения методов с понятными названиями

8. Типобезопасность

    Было: Использование примитивных типов напрямую

    Стало: Создание доменных моделей (TransportData, TransportResult)

9. Порядок членов класса

    Четкая структура: поля → конструкторы → методы

    Регионы для группировки связанных компонентов
